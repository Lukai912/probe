探针使用说明

1. 使用探针的必要条件

  1. 工程必须包含自定义 Application，并继承Application的onCreate()方法, 而且在AndroidManifest.xml文件中已经配置完成。例如：

    public class MyApplication extends Application {

@Override
public void onCreate() {
    // TODO Auto-generated method stub
    super.onCreate();
}
}

  2. 工程必须包含自定义的基类BaseActivity，并且工程中的所有Activity继承BaseActivity。BaseActivity 继承onResume()，onPause()，onDestroy()方法，并且在AndroidManifest.xml文件中已经配置完成。例如：

    public class BaseActivity extends Activity {

@Override
protected void onResume() {
    // TODO Auto-generated method stub
    super.onResume();
}

@Override
protected void onPause() {
    // TODO Auto-generated method stub
    super.onPause();
}

@Override
protected void onDestroy() {
    // TODO Auto-generated method stub
    super.onDestroy();
}
}

2. 探针在 AndroidStudio 中的使用配置

在AndroidStudio中，通过使用aar包的形式使用探针的相关功能。

使用步骤：

  1. 在module目录下创建libs 目录，将我们提供的探针 probe-debug.aar放入其中

  2. 在module的build.gradle中添加如下信息:

    android{
     repositories{
            flatDir{
                dirs 'libs'
            }
        } 
   } 

dependencies {
    compile (name:'probbugtags-release',ext:'aar')
}

  3. 在自定义的 Application 的 onCreate() 方法中，加入如下代码：

    ApplicationInit.onCreateInit(this);

    并导入类：

    import com.csmijo.probbugtags.ApplicationInit;

  4. 在自定义的 BaseActivity的 onCreate()，onPause()，onDestroy()方法中，分别加入如下代码：

    BugTagAgent.onResume(this);

BugTagAgent.onPause(this);

BugTagAgent.onDestory(this);

    并导入类：

    import com.csmijo.probbugtags.BugTagAgent;

  5. 在module的AndroidManifest.xml文件中，加入 UMS_APPKEY，例如：

    <meta-data
        android:name="UMS_APPKEY"
        android:value="58fde063a263***d64faa7ced0cbcce9"/>
